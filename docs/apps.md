# Django Apps

## Active Apps

### `users/`
Custom user model (`users.User`) with email as the primary identifier.

**Models:**
- `User` — extends `AbstractUser`; fields: `is_platform_admin`, `is_email_verified`, `auth_provider`, `login_count`, `last_login_ip`
- `VCSAccount` — stores encrypted OAuth tokens for GitHub/GitLab (used for VCS operations, not login)

**Key views:** login, logout, register (creates org), email verification, invite acceptance, VCS disconnect

---

### `organizations/`
Multi-tenancy layer. Every resource in the app belongs to an Organization.

**Models:**
- `Organization` — name, slug, owner (FK User), is_active
- `OrganizationMembership` — user + org + role (`SUPER_ADMIN` / `ADMIN` / `MANAGER` / `USER`), invited_by
- `OrganizationInvite` — email, org, role, UUID token (expires 7 days)

**Key views:** org select/switch, member list, role change, invite/remove member

---

### `subscriptions/`
Gates features by plan. Enforced via middleware (`SubscriptionMiddleware`) on every request.

**Models:**
- `Plan` — type (`STARTER` / `GROWTH` / `ENTERPRISE`), price, `max_instances`, `max_backups_per_month`, flags for staging/version-upgrade
- `Subscription` — org (1:1), plan, status (`ACTIVE` / `TRIAL` / `PAST_DUE` / `CANCELLED` / `SUSPENDED`), period dates, grace-period logic
- `UsageRecord` — tracks per-org usage events (BACKUP, STAGING, UPGRADE)

---

### `cloud/`
Manages connections to infrastructure providers.

**Models:**
- `ExternalServer` — bare-metal/VPS accessed via SSH; fields: host, port, username, auth_type, encrypted_password; tracks `is_verified`, `is_prepared`, `preparation_status`
- `CloudAccount` — DO or AWS credentials (all encrypted); `is_verified` flag, last verified timestamp
- `CloudServer` — a VM provisioned through a cloud account (status: PENDING/PROVISIONING/RUNNING/STOPPED/FAILED/DELETED)
- `SystemSSHKey` — singleton Ed25519 keypair generated by DafeApp; public key shown to users to add to their servers

**Key views:** add/verify external server, prepare server, add/verify cloud account, provision/destroy droplet, show public SSH key

---

### `deployments/`
Core app — orchestrates Odoo server provisioning and instance creation.

**Models:**
- `Infrastructure` — links an org to either an ExternalServer or CloudAccount; type: `PYOS` or `MANAGED`
- `Instance` — a provisioned cloud VM (Terraform output); tracks IP, status, Terraform state path
- `TerraformRun` — one execution of `terraform apply/destroy`; logs command, stdout, stderr, status
- `OdooServer` — a configured Odoo-ready server; fields: odoo_version, region, size, ip, dns_domain, port range, capacity, `is_reachable`, status (`PENDING` → `PROVISIONING` → `CONFIGURING` → `PROVISIONED` / `FAILED`)
- `OdooInstance` — one Odoo tenant on a server; fields: db_name, domain, http_port, systemd_service, nginx_site, ssl_enabled, status

**Key Celery tasks (`deployments/tasks.py`):**
- `provision_odoo_server` — runs Terraform (MANAGED) or skips it (PYOS), then runs Ansible
- `configure_odoo_server` — runs `setup_odoo_server_bare.yml` Ansible playbook
- `create_odoo_instance` — runs `create_odoo_instance_direct.yml` (direct IP:port) or domain-based playbook
- `delete_odoo_instance` — runs `delete_odoo_instance_direct.yml`, drops DB, cleans UFW
- `check_server_connectivity` — periodic beat task (every 2 min), pings all OdooServers

---

### `audit/`
Immutable event log for compliance and debugging.

**Model:**
- `AuditLog` — user, org, action (26+ action types), description, ip_address, user_agent, timestamp, metadata (JSON)

Action types include: LOGIN, LOGOUT, REGISTER, EMAIL_VERIFY, PASSWORD_CHANGE, SERVER_ADD, SERVER_VERIFY, CLOUD_ACCT_ADD, DROPLET_PROVISION, VCS_CONNECT, and more.

---

### `core/`
Houses the dashboard UI and shared abstract base classes.

**Abstract models:**
- `OrganizationScopedModel` — base for org-filtered models
- `OrganizationScopedManager` — `.for_org(org)` queryset helper

**Key views:** Dashboard home, Connections page, Profile, User management, Audit log viewer, VCS management, Installation docs
