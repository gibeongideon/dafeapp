Odoo Server Installation

Deploy Odoo directly on Ubuntu 22.04 / 24.04 via SSH — works with DigitalOcean Droplets, AWS EC2, and any PYOS server added to DafeApp.

Bare-metal (Ubuntu) DigitalOcean AWS EC2 Odoo 18 / 19
{% for t in tabs %} {% endfor %}

Path A — Via DafeApp UI

DafeApp provisions the server (Terraform) and prepares the Odoo environment (Ansible) automatically. Best for managed cloud accounts on DigitalOcean or AWS.

  1. 1.Add a Cloud Account (DO or AWS) under Connections.
  2. 2.Create an Infrastructure record (type: Managed).
  3. 3.Create an Odoo Server — DafeApp runs Terraform + Ansible to prepare the environment.
  4. 4.Add Odoo Instances to the provisioned server.
Go to Servers →

Path B — Manual CLI (SSH)

Use scripts/deploy_bare.sh to prepare an Odoo environment directly on any SSH-accessible server in one command. Add --standalone only if you explicitly want the legacy 8069 service too.

  1. 1.Spin up an Ubuntu 22.04 or 24.04 server.
  2. 2.Run deploy_bare.sh --ip <IP> --key <KEY>.
  3. 3.In DafeApp, use the PYOS form to provision the existing server in one step.

SSH Key Setup

Generate a key if you don't have one, then add the public key to your server.

# Generate a new key pair

ssh-keygen -t ed25519 -C "dafeapp-odoo" -f ~/.ssh/dafeapp_odoo

 

# Copy public key to server (password login must be enabled first)

ssh-copy-id -i ~/.ssh/dafeapp_odoo.pub ubuntu@<SERVER_IP>

 

# Or paste the public key when creating the server in your cloud console

cat ~/.ssh/dafeapp_odoo.pub

Quick Start — scripts/deploy_bare.sh

One script prepares an Odoo environment on any SSH-accessible Ubuntu server. Add --local to run the same installer on this machine for phase-one testing without Docker, or --standalone if you want the legacy 8069 service too.

Minimal (community, no nginx)

./scripts/deploy_bare.sh \

  --ip 165.22.100.50 \

  --key ~/.ssh/dafeapp_odoo

Simulation guide

# Local host simulation

sudo ./scripts/deploy_bare.sh \

  --local \

  --fresh \

  --version 19

# Then create instances on localhost with:

sudo ansible-playbook infra/ansible/create_odoo_instance_direct.yml \

  -i localhost, -c local \

  --extra-vars "odoo_version=19 db_name=app1 instance_name=app1 http_port=8070"

# SSH / remote simulation

./scripts/deploy_bare.sh \

  --ip 168.144.24.219 \

  --user root \

  --key ~/.ssh/id_rsa \

  --version 19

# If you use the UI, create the PYOS server from the single form in one step.

Local test (no SSH, no Docker)

./scripts/deploy_bare.sh \

  --local \

  --version 19

Fresh local test (wipe old host state first)

./scripts/deploy_bare.sh \

  --local \

  --fresh \

  --version 19

Use --fresh for repeat local tests so stale /odoo state does not block the installer. For SSH tests, omit --local and point the command at your remote host. Add --standalone only when you intentionally want a running Odoo service on 8069.

Instance playbooks also bootstrap PostgreSQL now: they create a default cluster if needed, start it, wait for pg_isready, and create the Odoo role before database creation. This avoids the old socket and missing-role failures during multi-instance setup.

With Nginx + SSL (Let's Encrypt)

./scripts/deploy_bare.sh \

  --ip 165.22.100.50 \

  --user ubuntu \

  --key ~/.ssh/dafeapp_odoo \

  --version 19 \

  --domain odoo.example.com \

  --email admin@example.com \

  --standalone

DO

DigitalOcean Setup

Step 1 — Generate a Personal Access Token

Go to DigitalOcean Console → API → Personal Access Tokens → Generate New Token. Enable Write scope.

# Add to .env

DIGITALOCEAN_TOKEN=dop_v1_xxxxxxxxxxxxxxxxxxxxxxxx

Step 2 — Configure DafeApp

# .env

DIGITALOCEAN_TOKEN=dop_v1_xxxxxxxx

TERRAFORM_SERVER_MODULE_DIR=/path/to/dafeapp/infra/terraform/odoo_server

ANSIBLE_ODOO_SERVER_PLAYBOOK=/path/to/dafeapp/infra/ansible/setup_odoo_server_bare.yml (optional override)

TERRAFORM_SSH_USER=ubuntu

TERRAFORM_SSH_KEY_PATH=~/.ssh/dafeapp_odoo

 

# Optional: DNS auto-creation

DNS_PROVIDER=digitalocean

DNS_ROOT_DOMAIN=example.com

DNS_CREATE_HOOK_CMD=/path/to/dafeapp/scripts/create_dns_record.sh

Step 3 — Recommended Droplet Sizes

Slug vCPU RAM Disk Use case
s-2vcpu-4gb24 GB80 GBDev / test
s-4vcpu-8gb48 GB160 GBSmall production
s-8vcpu-16gb816 GB320 GBMedium production

Step 4 — Manual CLI (no DafeApp)

# Create a droplet via doctl first, then deploy

doctl compute droplet create odoo-prod \

  --size s-4vcpu-8gb --image ubuntu-24-04-x64 \

  --region nyc3 --ssh-keys <KEY_ID>

 

DROPLET_IP=$(doctl compute droplet get odoo-prod --format PublicIPv4 --no-header)

 

./scripts/deploy_bare.sh \

  --ip "$DROPLET_IP" --key ~/.ssh/dafeapp_odoo \

  --version 19 --domain odoo.example.com --email admin@example.com

AWS EC2 Setup

Step 1 — IAM Credentials

Create an IAM user (or role) with EC2 full access. For Terraform you also need VPC read permissions.

# Add to .env

AWS_ACCESS_KEY_ID=AKIAXXXXXXXXXXXXXXXX

AWS_SECRET_ACCESS_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

AWS_DEFAULT_REGION=us-east-1

Step 2 — Configure DafeApp

# .env

AWS_ACCESS_KEY_ID=AKIAXXXXXXXXXXXXXXXX

AWS_SECRET_ACCESS_KEY=xxxxxxxx

AWS_DEFAULT_REGION=us-east-1

TERRAFORM_SERVER_MODULE_DIR=/path/to/dafeapp/infra/terraform/odoo_server

ANSIBLE_ODOO_SERVER_PLAYBOOK=/path/to/dafeapp/infra/ansible/setup_odoo_server_bare.yml (optional override)

TERRAFORM_SSH_USER=ubuntu

TERRAFORM_SSH_KEY_PATH=~/.ssh/dafeapp_odoo.pem

 

# Optional: Route53 DNS

DNS_PROVIDER=route53

AWS_ROUTE53_ZONE_ID=Z1XXXXXXXXXXXXXXXX

DNS_CREATE_HOOK_CMD=/path/to/dafeapp/scripts/create_dns_record.sh

Step 3 — Recommended Instance Types

Type vCPU RAM Use case
t3.medium24 GBDev / test
t3.large28 GBSmall production
t3.xlarge416 GBMedium production
m5.xlarge416 GBHigh-traffic production

Step 4 — Manual CLI (no DafeApp)

# Launch EC2 instance via AWS CLI

aws ec2 run-instances \

  --image-id ami-0c7217cdde317cfec \

  --instance-type t3.large \

  --key-name dafeapp_odoo \

  --security-group-ids sg-xxxxxxxxxx \

  --region us-east-1 \

  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=odoo-prod}]'

 

EC2_IP=$(aws ec2 describe-instances \

  --filters "Name=tag:Name,Values=odoo-prod" "Name=instance-state-name,Values=running" \

  --query "Reservations[0].Instances[0].PublicIpAddress" --output text)

 

./scripts/deploy_bare.sh \

  --ip "$EC2_IP" --key ~/.ssh/dafeapp_odoo.pem \

  --version 19 --domain odoo.example.com --email admin@example.com

CF

Cloudflare Platform Domain Setup

Step 1 — Add your shared platform domain to Cloudflare

Put the base domain you want DafeApp to manage in Cloudflare, for example dafeapp.com. New instances will then get automatic hostnames like app1.dafeapp.com.

Step 2 — Create a scoped API token

Create a Cloudflare API token limited to the shared platform zone only. The token should be able to read the zone and create or update DNS records for it.

# Recommended minimum permissions

Zone Read

Zone DNS Write

# Scope

Include → Specific zone → dafeapp.com

Step 3 — Copy the Cloudflare Zone ID

Open the dafeapp.com zone in Cloudflare and copy its Zone ID. DafeApp uses that together with the API token to create exact per-instance DNS records automatically.

Step 4 — Configure DafeApp environment variables

# Shared DafeApp domain

PLATFORM_BASE_DOMAIN=dafeapp.com

PLATFORM_DNS_PROVIDER=CLOUDFLARE

PLATFORM_DNS_API_TOKEN=cf_xxxxxxxxxxxxxxxxxxxxx

PLATFORM_DNS_ZONE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

PLATFORM_DNS_PROXIED=False

 

# TLS / Traefik

TRAEFIK_DEFAULT_TLS_MODE=LETS_ENCRYPT

TRAEFIK_ACME_EMAIL=admin@example.com

Start with PLATFORM_DNS_PROXIED=False while validating routing and Let's Encrypt. You can enable Cloudflare proxying later once the shared domain flow is stable.

Step 5 — Open ports 80 and 443 on your server

Traefik and Let's Encrypt need public HTTP and HTTPS access. Without ports 80 and 443, automatic SSL for the DafeApp domain will not complete.

Custom domains still work separately

Customers do not need to use your Cloudflare account for their own domains. They can create an A record at their own provider pointing their hostname to your server IP, and DafeApp can attach that hostname as an additional alias while keeping the default *.dafeapp.com address active.
